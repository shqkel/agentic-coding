# Code Reviewer Skill

이 스킬은 코드의 품질을 높이고 유지보수성을 강화하기 위한 전문적인 코드 리뷰 가이드를 제공합니다.

## 리뷰 핵심 원칙
1. **가독성 (Readability)**: 변수명, 함수명이 그 역할을 명확히 설명하는가?
2. **성능 (Performance)**: 불필요한 연산이나 중복된 데이터베이스 쿼리가 없는가?
3. **보안 (Security)**: 민감 정보 노출, SQL Injection 등 보안 취약점이 없는가?
4. **일관성 (Consistency)**: 프로젝트의 기존 코딩 스타일 및 컨벤션을 따르는가?
5. **설계 (Design)**: 함수가 하나의 기능만 수행하며(SRP), 의존성 관리가 적절한가?

## 체크리스트 (참조용)
상세한 언어별/도메인별 체크리스트는 [references/checklist.md](references/checklist.md)를 참고하세요.

## 실행 가이드
1. 변경된 코드(`git diff` 또는 파일 내용)를 분석합니다.
2. 각 원칙에 따라 개선이 필요한 부분을 식별합니다.
3. 비판이 아닌 **개선 제안(Suggestions)** 형태로 피드백을 작성합니다.
4. 가능한 경우 개선된 코드 예시를 함께 제공합니다.
